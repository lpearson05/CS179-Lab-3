CS179 Lab 3: Exercise #2 - jQM Instructions

Goal: Get comfortable parsing and reading JS object definitions and JSON.

Instructions

The only file you need to modify is lab3/exercise-2/exercise.js. See the -pseudocode version for help. Also see w3school's array object reference .

    1. Process and display tweets.
    1. Process the tweet objects appropriately in displayTweets. At least, print the tweet text. (Hint: Examine the results being logged to the console in the ajax call to Twitter's api.) To print out the tweets in a more "sensible" manner, you will have to first analyze the tweet objects in the console.
    2. Construct and return a li representation of the tweet in buildTweet.
      • Add or style the list item as you wish, but at least put the tweet text in it.
      • *Make the tweets look more like tweets by changing the color of references and hashes* (Advanced)
    3. Add the newly-constructed tweets to the list.
    4. Limit the list to a certain length by deleting old tweets.

    For help, see demo #2, which introduces jQueryMobile. See the end of the presentation as well. Also check out the jQM Docs on Lists.

    2. Store recent queries by filling the body of saveQuery and making clicking on the items resubmit the form with the stored query string.
    1. Make a new clickable li containing the string.
    2. Bind to it's click event a function that resubmits form#query-form with the value of input#query-input set to the it's text content.
    3. Add the li to ul#recent
    4. Remove old queries from the list and storage to keep it to a certain length.
    5. Refresh the list.
    3. Use session storage to persist stored queries.
    1. Write the body of storeQuery to store queries in sessionStorage.
    2. Write the body of restoreQueries to call storeQuery on each item in client-side storage.

    For help, see demo #3, which shows different client-side storage methods.
